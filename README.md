# Localize Service

This project is a Node.js application with Elasticsearch integration. Below are the instructions for setting up and running the application, along with testing procedures.

## Prerequisites

- [Docker](https://www.docker.com/products/docker-desktop) installed on your machine.
- [Node.js](https://nodejs.org/) and [npm](https://www.npmjs.com/) installed on your machine.

## Installation

### 1. Set up Elasticsearch

To set up Elasticsearch using Docker, run the following command:

```bash
$ docker-compose up -d
```

For more details on Elasticsearch, refer to the [Elasticsearch documentation](https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-multi-match-query.html).

### 2. Install Node.js modules

Navigate to the project directory and install the necessary Node.js modules:

```bash
$ npm install
```

## Running the Application

You can run the application in different modes:

### Development Mode

To start the application in development mode, run:

```bash
$ npm run start
```

### Watch Mode

For watch mode, which automatically restarts the server on file changes, run:

```bash
$ npm run start:dev
```

### Production Mode

To start the application in production mode, run:

```bash
$ npm run start:prod
```

### API Documentation

Once the application is running, you can access the API documentation generated by Swagger at:

```
http://localhost:8000/api
```

## Testing

This project includes various testing scripts:

### Unit Tests

To run unit tests, execute:

```bash
$ npm run test
```

### End-to-End (E2E) Tests

To run end-to-end tests, execute:

```bash
$ npm run test:e2e
```

### Test Coverage

To check the test coverage, execute:

```bash
$ npm run test:cov
```

## Additional Information

- Ensure Docker is running before starting the backend service.
